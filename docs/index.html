🤖 Agent 深度验证

什么是 Agent Loop？

一个受 Webwright 启发的自主验证模式，其中 LLM 代理编写并执行 Playwright 脚本来深度验证网站。代理会迭代地：

✍️

1. 编写脚本

基于任务描述编写 Playwright 脚本

⚡

2. 执行脚本

执行脚本并捕获输出和截图

🔧

3. 自动修复

检查结果并自动修复问题

✅

4. 验证完成

通过自我反思验证任务完成

新增 CLI 命令

命令 描述 示例
verify:deep Agent深度验证 npm run verify:deep -- --url http://example.com --task "检查页面"
verify:craft 生成可复用脚本 npm run verify:craft -- --url http://example.com --task "验证导航"

verify:deep 选项

  • --config, -c — 站点配置文件
  • --url, -u — 目标 URL
  • --task, -t — 任务描述（必需）
  • --model, -m — LLM 模型（默认：gpt-4）
  • --max-steps, -s — 最大步数（默认：30）
  • --api-key, -k — API 密钥
  • --api-base, -b — API 基础 URL

⚡ 快速+深度验证编排

什么是编排验证？

编排层将快速验证（Verifier）和深度验证（Agent Loop）串联成一个自动化流水线，智能决策是否需要深度验证，节省 Token 成本。

工作流程

1️⃣

快速检查

对每个站点运行快速检查（状态码、性能、可访问性、SEO、控制台）

2️⃣

结果分析

分析检查结果，判断是否需要深度验证

3️⃣

智能决策

所有检查通过 → 跳过深度验证，节省Token

4️⃣

自动诊断

有检查失败 → 自动生成诊断任务 → 触发Agent深度验证

5️⃣

深入诊断

Agent深入诊断并尝试自动修复问题

6️⃣

统一报告

输出统一报告：快速验证 + 深度验证结果汇总

自动任务生成

编排器会从快速检查的失败信息自动生成深度验证任务描述（中文）：

任务映射规则 
// HTTP 状态码失败
"验证页面HTTP状态码和导航是否正常"

// 可访问性问题
"检查页面可访问性问题，特别是[具体错误]"

// 性能指标异常
"分析页面性能指标，[具体指标]"

// 控制台错误
"调查控制台错误和警告：[具体错误]"

CLI 命令

Command Description
verify:orchestrated 快速+深度编排验证

使用示例

Bash 
# 基础编排验证
npm run verify:orchestrated -- --config sites/quick-check.json

# 严格模式：非关键失败也触发深度验证
npm run verify:orchestrated -- --config sites/quick-check.json --strict

# 指定深度验证模型
npm run verify:orchestrated -- --config sites/quick-check.json --deep-model glm-5.1

# 仅运行快速验证，跳过深度验证
npm run verify:orchestrated -- --config sites/quick-check.json --skip-deep

选项说明

  • --config, -c — 站点配置文件（必需）
  • --strict, -s — 非关键失败也触发深度验证
  • --deep-model, -m — LLM模型（默认: gpt-4o）
  • --output, -o — 输出报告路径
  • --json, -j — JSON格式输出
  • --skip-deep — 仅运行快速验证

⚙️ 配置文件格式

基础配置项

  • name — 站点名称（用于报告标识）
  • url — 目标URL地址
  • expectedStatusCode — 期望的HTTP状态码（默认200）

视口配置

  • viewport — 单一视口尺寸 {width, height}
  • viewports — 多分辨率数组 [{name, width, height}, ...]

检测配置

  • screenshots — 截图类型数组: ["fullpage", "viewport"]
  • checks — 启用的检测: ["screenshot", "console", "performance", "accessibility", "seo"]
  • customChecks — 自定义检测逻辑（高级用法）
  • timeout — 页面加载超时时间（毫秒）

🔧 API / 编程式使用

TypeScript 
import { Verifier } from 'e2e-verifier';

const config = {
  name: 'My Site',
  url: 'https://example.com',
  expectedStatusCode: 200,
  checks: ['screenshot', 'console', 'performance']
};

const verifier = new Verifier(config);
const result = await verifier.verify();
console.log(result.passed ? '✅ PASSED' : '❌ FAILED');

🎯 检测类型参考

📸

screenshot

全页面/视口截图，支持多分辨率自动生成，自动处理页面滚动和元素等待。

💻

console

控制台错误/警告检测，捕获 JavaScript 异常、API 失败和资源加载问题。

⚡

performance

核心性能指标：FCP、LCP、加载时间、页面重量、资源数量等。

♿

accessibility

ARIA 标签、对比度、语义化 HTML 检测，确保无障碍访问标准。

🔍

seo

SEO 基础检测：title、meta 描述、h1 标签、Open Graph 标签、favicon 等。

🛠️

customChecks

自定义检测逻辑，支持任意 JavaScript 验证规则和断言。
